1. Разбивка на компоненты выполнена для удобста переиспользования кода и интеграции в проект
2. Для правильной интеграции лучше смотреть на использование компонента в шаблоне страницы
3. В каждом компоненте подробно прописаны комментарии
4. Существует вероятность, что компонент мог быть не обновлен в этой папке, но обновлен в проекте. Поэтому лучше брать стили, верству и скрипты из проекта, а в папке компонентов опираться на комментарии для понимания структуры
5. В папке Basic лежат стили, которые используются в каждом компоненте и на каждой странице. По ходу верстки будут обновляться вместе с обновлением списка компонентов
6. Стили на каждой странице, как и скрипты, лучше подключать из разных файлов, чтобы не было пересечений (style.css и script.js - единые файлы для каждой страницы, остальные индивидуальны)
